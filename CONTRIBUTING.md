# Contributing to nuclei-wordfence-cve

Thank you for your interest in contributing to the **nuclei-wordfence-cve** project! Your contributions help improve this project and benefit the wider community by enhancing WordPress security scanning capabilities.

## How to Contribute

### Reporting Issues
If you encounter any bugs, glitches, or have suggestions for improvement, please open an issue on the [GitHub Issues](https://github.com/topscoder/nuclei-wordfence-cve/issues) page. When reporting an issue, please include as much relevant detail as possible, such as:

- Steps to reproduce the issue
- Expected vs actual behavior
- Your environment details (Nuclei version, OS, etc.)
- Any error messages or logs

### Submitting Pull Requests
If you'd like to contribute code, please follow these steps:

1. **Fork the repository** to your GitHub account.
2. Clone your fork locally: ```git clone https://github.com/your-username/nuclei-wordfence-cve.git```
3. **Create a new branch** for your feature or bugfix: ```git checkout -b my-feature-branch```
4. Make your changes with clear and concise commit messages.
5. Test your changes thoroughly to ensure they work as expected.
6. Push your branch to your fork: ```git push origin my-feature-branch```
7. Open a **Pull Request** from your branch to the main repositoryâ€™s `main` branch.

### Contribution Guidelines
- Please ensure your code follows consistent style and best practices.
- Include comments and documentation for any complex or non-obvious changes.
- If adding new templates or updating existing ones, make sure they are tested against relevant WordPress versions and vulnerabilities.
- Avoid breaking existing functionality and respect backward compatibility.

### Code of Conduct
By participating in this project, you agree to abide by the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/0/code_of_conduct/). Please be respectful and considerate in all interactions.

## Getting Help
If you need help or have questions about contributing, feel free to open an issue or contact the maintainers.
